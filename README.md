# ctx

A powerful CLI tool for generating and applying context snapshots optimized for LLM interactions.

## Features

- **Snapshot Generation** (`ctx read`): Create repository snapshots in `@@CTX` format for AI context
- **Edit Application** (`ctx edit`): Apply LLM-generated edits safely with anchor-based matching
- **Smart Selection** (`ctx select`): Graph-based file selection using structural and co-change analysis
- **Interactive Wizard** (default): Guided setup when you run `ctx` with no arguments

## Installation

### One-Command Install

**macOS/Linux:**
```bash
curl -fsSL https://raw.githubusercontent.com/akash202k/ctx/main/scripts/install.sh | sh
```

**Windows (PowerShell):**
```powershell
iwr -useb https://raw.githubusercontent.com/akash202k/ctx/main/scripts/install.ps1 | iex
```

### Alternative: Go Install

If you have Go installed:
```bash
go install github.com/akash202k/ctx/cmd/ctx@latest
```

### Build from Source

```bash
git clone https://github.com/akash202k/ctx
cd ctx
make build
sudo mv ctx /usr/local/bin/  # or add to PATH
```

## Usage

### Interactive Wizard (Recommended)

Run the interactive wizard for guided setup:
```bash
ctx
```

The wizard walks you through:
- Choosing command (read/edit)
- Setting base path
- Adding include/exclude rules
- Configuring output options
- Setting verbosity

### Generate Repository Snapshot

```bash
# Interactive mode
ctx

# Or use flags directly:

# Copy snapshot to clipboard
ctx read

# Save to file
ctx read --output snapshot.ctx

# Include specific paths
ctx read --include internal --include cmd

# Exclude paths
ctx read --exclude testdata --exclude vendor

# Verbose output
ctx read --verbose
```

### Apply Edits

```bash
# Apply from clipboard
ctx edit

# Apply from file
ctx edit --input edits.ctx

# Specify base path
ctx edit --base-path /path/to/repo
```

### Smart File Selection

```bash
# Select files for a task
ctx select --task "fix authentication bug" --entry auth/login.go

# Adjust token budget
ctx select --task "add caching" --budget 16000

# JSON output
ctx select --task "refactor database layer" --json

# Control graph distance
ctx select --task "optimize queries" --distance 5
```


## Format Specification

The `@@CTX` format is designed for safe, streaming-compatible edits:

```
# CTX_GENERATED_FILE

@@CTX<META-INFO>
ABOUT_CTX_FILE: Generated by ctx
BASE_PATH: /path/to/repo
FILTER_RULES: [{"type":"include","path":"."}]
STRUCTURE:
├─ cmd
│  └─ ctx
│     └─ main.go
└─ internal
   └─ snapshot
      └─ snapshot.go
@@CTX</META-INFO>

@@CTX<FILE path="cmd/ctx/main.go">
package main

import "fmt"

func main() {
    fmt.Println("Hello, ctx!")
}
@@CTX</FILE>
```

### Edit Format

```
@@CTX<FILE path="cmd/ctx/main.go">
package main

import (
    "fmt"
    "os"
)
@@CTX<NO-CHANGE />
func main() {
    if len(os.Args) < 2 {
        fmt.Println("Usage: ctx <command>")
        return
    }
    fmt.Println("Hello, ctx!")
}
@@CTX</FILE>
```

## Architecture

- **Snapshot Layer** (`internal/snapshot`): Repository walking and `@@CTX` format generation
- **Editor Layer** (`internal/editor`): Parse and apply edits with anchor-based matching
- **Graph Layer** (`internal/graph`): Dependency graph construction from AST parsing
- **Selection Layer** (`internal/score`): Relevance scoring using graph distance, co-change, and recency
- **Walker Layer** (`internal/walker`): File enumeration with `.gitignore` support

## Comparison with Astrolark

`ctx` is a complete rewrite of Astrolark in Go:

| Feature | Astrolark (TypeScript) | ctx (Go) |
|---------|------------------------|----------|
| Format | `@@ALK` | `@@CTX` |
| Language | Node.js/TypeScript | Go |
| CLI | yargs | Cobra |
| Performance | ~1s for 100 files | ~100ms for 100 files |
| Graph Analysis | ❌ | ✅ (import/call graph + git co-change) |
| Smart Selection | ❌ | ✅ (graph-based ranking) |
| Binary Size | ~50MB (with Node.js) | ~8MB (static binary) |
